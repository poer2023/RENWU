🔧 连线功能调试与修复报告

🔍 问题分析

主要问题：手动连线功能无法正常工作，连接端点不显示

发现的问题：

1. 连接端点显示问题 已修复
问题：ConnectionPorts 组件有内部的 v-show="visible" 控制，但 TaskCard 没有传递 visible prop 修复：将 TaskCard 中的 v-show 改为 :visible prop 传递
2. 事件传递链条 正常
事件流：ConnectionPorts → TaskCard → TaskRenderer → StickyCanvas → useConnections 所有环节的事件绑定都正确
3. 连接逻辑 完整
useConnections 实现了完整的拖拽连线逻辑： - startConnection: 开始连线 - handleConnectionDrag: 拖拽跟踪 - handleConnectionEnd: 结束连线并创建依赖

🛠️ 修复措施

修复连接端点显示：
// 在 TaskCard.vue 中 // 修改前： <ConnectionPorts v-show="isSelected || isConnecting" :is-connecting="isConnecting" /> // 修改后： <ConnectionPorts :visible="isSelected || isConnecting" :is-active="isConnecting" @start-connection="startConnection" />

🧪 测试步骤

1. 打开 TaskWall 应用（下方 iframe）
2. 点击选择任一任务卡片
3. 查看任务卡片四周是否出现蓝色连接点（上、右、下、左）
4. 按住鼠标左键拖拽任一连接点
5. 拖拽到另一个任务卡片上释放鼠标
6. 检查是否成功创建了连线
