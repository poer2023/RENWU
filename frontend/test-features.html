🧪 TaskWall 功能测试指南

验证删除功能、AI生成子任务和快捷键功能

打开 TaskWall 应用 查看 API 文档

🗑️ 任务删除功能测试

测试步骤:

  1. 在 TaskWall 中创建一个测试任务
  2. 点击任务卡片选中它
  3. 点击任务卡片右下角的删除按钮 (🗑️)
  4. 在确认对话框中点击"删除"按钮
预期结果:

任务应该被成功删除，从画布中消失，同时相关的连线也会被清理。

快捷键删除测试:

  1. 选中一个任务卡片
  2. 按下 Delete 键或 Backspace 键
  3. 或者按下 Ctrl+D (Windows) / Cmd+D (Mac)
  4. 确认删除
预期结果:

任务应该通过快捷键被删除，功能与点击删除按钮相同。

🤖 AI生成子任务功能测试

⚠️ 前提条件:

确保后端已配置有效的 Gemini API Key。可以在设置中配置，或通过环境变量 GEMINI_API_KEY 设置。

测试步骤:

  1. 创建一个有明确标题和描述的主任务，如："开发用户登录功能"
  2. 点击任务卡片选中它
  3. 点击任务卡片底部的 "⚡" 生成子任务按钮
  4. 等待 AI 处理（可能需要几秒钟）
预期结果:

1. AI 应该生成 3-5 个相关的子任务
2. 子任务会自动创建在父任务右侧，垂直排列
3. 每个子任务都会自动连线到父任务
4. 子任务具有合理的标题、描述和优先级
5. 控制台会显示详细的创建日志

快捷键生成子任务:

  1. 选中一个任务卡片
  2. 按下 Ctrl+G (Windows) / Cmd+G (Mac)
  3. 等待 AI 处理
预期结果:

功能与点击生成子任务按钮相同。

⌨️ 其他快捷键功能测试

编辑任务快捷键:

  1. 选中一个任务卡片
  2. 按下 Enter 键或 F2 键
预期结果:

任务进入编辑模式，可以修改标题和描述。

🔍 调试和故障排除

检查控制台日志:

  1. 打开浏览器开发者工具 (F12)
  2. 切换到 Console 标签
  3. 执行各种操作时观察日志输出
  4. 查看是否有错误信息
应该看到的日志:

• 删除任务: "Task deleted successfully" 等
• 生成子任务: "开始生成子任务...", "收到AI生成的子任务:", "子任务创建成功:" 等
• 创建连线: "创建依赖连线成功:" 等

API 状态检查:

  1. 访问 http://localhost:8765/docs
  2. 测试 DELETE /tasks/{task_id} 端点
  3. 测试 POST /ai/subtasks 端点
  4. 测试 POST /dependencies/ 端点

🎯 完整测试流程

端到端测试:

  1. 创建一个主任务："开发电商网站"
  2. 为该任务生成子任务（Ctrl+G）
  3. 验证子任务自动连线
  4. 选择一个子任务并删除（Delete键）
  5. 验证连线也被清理
  6. 再次生成新的子任务
  7. 检查所有功能是否正常工作
预期结果:

整个流程应该顺畅运行，没有错误，UI响应及时，连线准确。
